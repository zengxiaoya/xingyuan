# 星渊 · Claude Code 实现指令

你是一个全栈开发工程师，请根据以下完整设计方案实现「星渊」Web应用。

---

## 技术栈

- **前端**：React 18 + Vite + React Router v6 + Zustand
- **后端**：Node.js + Express（ESM模块）
- **AI**：Claude API（claude-sonnet-4-20250514），流式输出
- **样式**：纯 CSS + CSS变量，不使用 UI 库
- **字体**：Orbitron（英文标题）+ Noto Serif SC（证书宋体）via Google Fonts
- **部署**：Docker + docker-compose，Nginx 反向代理

## API 接口规范

### POST /api/chat（SSE流式）
```js
// 请求体
{ messages: [{role, content}], context: { type: 'level'|'scientist'|'general', id: 'solar_lv1' } }
// 响应：text/event-stream
// data: {"text": "..."} ... data: [DONE]
```

### POST /api/quiz
```js
// 请求体
{ levelId: 'solar_lv1', type: 'choice'|'guess'|'creative' }
// 响应体（choice）
{ question, options: [A,B,C,D], answer: 0, explanation }
// 响应体（guess）
{ riddle, hint, keywords: ['关键词'] }
// 响应体（creative）
{ question, prompt }
// 失败时前端自动使用 levels.js 中的 fallbackQuestions
```

### POST /api/evaluate
```js
// 请求体
{ levelId, question, answer }
// 响应体
{ feedback: 'NOVA个性化反馈...' }
// 失败时返回：{ feedback: '你的想象力太棒了！NOVA为你骄傲！' }
```

## 全局状态流转

```
Onboarding 完成 → setUser() → 跳转 /map
/map 点击关卡节点 → 跳转 /level/:id
Level 通关 → completeLevel() → 解锁徽章+科学家 → 返回 /map
任意页面点击 NOVA → openNova(context) → 全屏弹出对话
/achievement 点击生成证书 → 跳转 /certificate
```

---

## 关卡页状态机

```
phase: 'q1' | 'q2' | 'q3' | 'complete'
qState: 'idle' | 'correct' | 'wrong'

idle → 用户答题 → correct（显示粒子爆炸+全屏闪现）
                → wrong（卡片抖动+全屏提示）
correct/wrong → 点击"下一题" → 进入下一个 phase
q3 answered → complete → 调用 completeLevel()
```

### nginx/nginx.conf 关键配置
```nginx
# SSE流式必须关闭缓冲，否则NOVA打字机效果不工作
location /api {
  proxy_pass http://server:3001;
  proxy_buffering off;
  proxy_cache off;
  proxy_read_timeout 120s;
}
# React Router需要回退到index.html
location / {
  proxy_pass http://client:80;
  try_files $uri $uri/ /index.html;
}
```

## 注意事项

1. API Key 只存在服务器 .env 文件，通过 docker-compose 环境变量注入，不写入代码
2. SSE 流式响应：nginx 必须设置 `proxy_buffering off`
3. React Router：nginx 必须配置 `try_files $uri /index.html` 回退
4. localStorage key：用户信息用 `xingYuan_user`，进度用 `xingYuan_progress`
5. 降级策略：quiz/evaluate 接口失败时使用 fallbackQuestions，不影响闯关主流程
6. CentOS 7 SELinux：容器挂载目录报权限时执行 `setenforce 0`
